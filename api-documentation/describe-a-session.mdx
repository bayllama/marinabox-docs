## Get Session Details

Retrieves detailed information about a specific active session.

### Parameters

<ParamField path="session_id" type="string" required>
  The unique identifier of the session to retrieve
</ParamField>

### Response

<ResponseField name="session_id" type="string">
  Unique identifier for the session
</ResponseField>

<ResponseField name="container_id" type="string">
  Docker container ID associated with the session
</ResponseField>

<ResponseField name="vnc_port" type="integer">
  Port number for VNC connection
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of session creation
</ResponseField>

<ResponseField name="env_type" type="string">
  Type of environment (`browser` or `desktop`)
</ResponseField>

<ResponseField name="debug_port" type="integer">
  Debug port number (only for browser environment)
</ResponseField>

<ResponseField name="status" type="string">
  Current session status (`running`)
</ResponseField>

<ResponseField name="runtime_seconds" type="number">
  Current runtime duration in seconds
</ResponseField>

<ResponseField name="tag" type="string">
  Custom tag assigned to the session (if any)
</ResponseField>

### Example Request

```bash
curl "http://localhost:8000/api/sessions/abc123xyz"
```

### Example Response

```json
{
  "session_id": "abc123xyz",
  "container_id": "abc123xyz789",
  "vnc_port": 5002,
  "created_at": "2024-12-14T10:30:00Z",
  "env_type": "browser",
  "debug_port": 4002,
  "status": "running",
  "runtime_seconds": 3600,
  "tag": "Development"
}
```

### Error Responses

| Status Code | Description |
|------------|-------------|
| 404 | Session not found |
| 500 | Internal server error |
